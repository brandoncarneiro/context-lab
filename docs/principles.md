# Principles

## 1. Better AI output starts with cleaner context

The quality of ChatGPT's help depends heavily on whether it is using accurate, current, and relevant context.

## 2. Separate facts, inferences, and unknowns

Do not let the model collapse uncertainty into confidence. A useful assistant should show what it knows, what it infers, and what it does not know.

## 3. Do not over-save memory

Durable memory is powerful but dangerous. Tactical context goes stale fast and should usually live in a working context document instead.

## 4. Use dated working context

Operating profiles should include a `Last updated` date. Stale context should be easy to spot.

## 5. Make workflows inspectable before making them autonomous

A workflow should expose assumptions, inputs, outputs, risks, and approval points before it is automated.

## 6. Optimize for execution

Good AI output should produce decisions, drafts, plans, checklists, prompts, or next actions. Vague reflection is not enough.

## 7. Confirm memory execution

When memory changes are applied, the model should confirm what was saved, updated, forgotten, intentionally not saved, and what could not be changed.
