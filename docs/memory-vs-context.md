# Memory vs Working Context

Memory is for stable context.

Working documents are for live operating state.

## Save to memory when information is

- stable
- broadly useful
- not sensitive
- unlikely to become stale
- helpful across many unrelated conversations

## Keep in a working context document when information is

- current but temporary
- tactical
- project-specific
- confidential
- likely to change
- only relevant to a narrow workflow

## Do not save when information is

- speculative
- emotionally charged but temporary
- about active negotiations
- about private third parties
- stale
- too granular
- not useful across future conversations

## Examples

| Save to memory | Keep in context doc |
|---|---|
| Stable role | Current projects |
| Communication preferences | This quarter's goals |
| Durable interests | Active deals |
| Long-term constraints | Pipeline status |
| Persistent advice style | Weekly priorities |
| Tool preferences | Tactical plans |

## Rule of thumb

If it may be wrong in 30 to 90 days, do not save it as durable memory.
