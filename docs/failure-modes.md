# Common Failure Modes

## 1. Saving tactical details to memory

Bad: "Remember that I am working on this proposal this week."

Better: keep this in a working context document.

## 2. Letting ChatGPT infer too much

Bad: "Based on what you know about me, make a plan."

Better: "Separate confirmed facts from assumptions. Ask before relying on anything uncertain."

## 3. Treating old context as current

Bad: using project details from months ago without checking dates.

Better: add a `Last updated` date to every operating profile.

## 4. Creating a master prompt that is too long

Bad: pasting a 5,000-word master prompt into every conversation.

Better: use a short prompt for normal work and a detailed one for major strategy sessions.

## 5. Confusing preferences with identity

Bad: "Always be aggressive and contrarian."

Better: "Be direct and challenge weak assumptions, but calibrate tone to the stakes."

## 6. Asking too many questions

Bad: turning every task into an intake form.

Better: ask only questions that materially change the answer. Otherwise make a best-effort recommendation and state assumptions.

## 7. Omitting execution confirmation

Bad: silently claiming memory work is done without saying what changed.

Better: end with a concise execution or handoff confirmation that lists saved, updated, deleted, intentionally skipped, and unsupported changes.

## 8. Pretending memory was changed

Bad: saying memory changed when the chat does not have durable memory-management capability.

Better: check capability first. If unavailable, produce a Memory Execution Sheet and say no memory changes were executed.

If only some memory actions are supported, execute only the supported actions and list the rest as manual follow-up.
