# Contributing

Useful contributions include:

- new workflow prompts
- better examples
- improved memory/context guidelines
- role-specific adaptations
- failure modes and anti-patterns
- model-specific notes

Please avoid:

- hype
- vague productivity advice
- prompts that encourage over-saving memory
- prompts that rely on unsupported assumptions
- prompts that produce bloated outputs without execution value

## Contribution standard

A good contribution should make the workflow more inspectable, more practical, or safer against stale context.
