# Example Memory Cleanup

This optional founder/operator example is fictionalized. Adapt it for your own role or use case.

## Should be saved as durable memory

- "The user prefers direct, execution-focused advice that separates facts, assumptions, and unknowns."
- "The user works on a software or services business and values clear, inspectable workflows over vague advice."

## Should be kept only in this project/context document

- Current quarterly goals
- Active projects under development
- Launch plans
- Sales or partnership pipeline
- Specific investor, customer, client, or partner conversations

## Should not be saved because temporary

- Weekly priorities
- Draft messaging
- Active task lists
- One-off travel plans

## Should be deleted or corrected

- Any old role, organization, project, or status that is no longer current

## Sensitive context to use only when explicitly provided

- Customer, client, or partner names
- Contract or deal terms
- Financial numbers
- Private team issues
